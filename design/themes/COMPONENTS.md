# RimeConfig 0.2.0 · 组件规格（新增页面 / 弹窗 / 托盘）

> 配套 `TOKENS.md` 使用。所有组件颜色/字体均引用令牌，三套皮肤（A 浅色 / B 深色 / C 水墨）自动继承，不单独设色。
> 本轮仅设计，未接功能。

## 1. 符号表页（SymbolsPage）

布局：左侧分类列表（212px 卡片）+ 右侧符号表格，顶部搜索/新增/分类筛选。

| 元素 | 规格 |
|---|---|
| 分类列表项 | 高 ~38px，圆角 7；hover `--bg-hover`；选中 `--bg-active-nav` + `--accent` 文字；右侧计数胶囊 `--bg-elevated` 底 |
| 新建分类 | 虚线边框 `--border-strong`，文字 `--accent`，整行可点 |
| 符号芯片 | 每符号 30×30，圆角 6，`--bg-elevated` 底 + `--border` 描边，字号 15 |
| 表格列 | 键位（等宽字体）/ 符号（芯片组）/ 分类（标签）/ 操作（编辑·删除） |
| 未保存态 | 状态栏显示「● 有未保存的修改」用 `--conflict-text` 色，与词库页一致 |

## 2. 方案配置页（SchemaPage）

| 元素 | 规格 |
|---|---|

| 复选框（五态沿用现有 QSS） | 20×20 圆角 5；未选 `--border-strong`；选中 `--accent` 底 + 白勾；半选用减号；禁用态降透明度 |

## 3. 设置页（SettingsPage）

- 分区：常规 / 外观·主题 / 关于。
- 常规行：标签（左）+ 控件（右对齐）。控件含路径输入框（280px）、开关、键位芯片、下拉。
- 外观·主题：三张主题卡（见 `TOKENS.md` 第 6 节 + `settings_theme_switcher.png`），选中即实时预览。

## 6. 令牌继承说明（给开发）

- 所有上述组件在 QSS 中**只引用令牌色值**（来自 `TOKENS.md`），不写死主题色。
- 切换主题 = `theme.py` 换令牌 → 重设 `QApplication` 样式表；弹窗/托盘用 `QApplication` 继承，无需单独处理（与现有 `apply_theme()` 一致）。
- C 水墨的专属装饰（标题宋体、印章 Logo、左侧渐变条）由 `theme.py` 在切到 `ink` 时注入，切走移除（见 `application_ink.qss` 注释）。
- 组件状态（hover/active/disabled/focus）三套皮肤保持一致交互语言，仅配色随令牌变化。

## 8. 交互状态（空态 / 加载态 / 批量选择态）

> 本轮细化词库管理页的三种关键中间态，统一组件语言，三套皮肤令牌继承。源：`state_empty.html / state_loading.html / state_batch.html`。

### 8.1 空态 Empty（词库 0 条）
| 元素 | 规格 |
|---|---|
| 触发 | 词库管理页无任何词条时，表格体替换为空态区；顶部工具栏（搜索/新增）保留可用 |
| 插画 | 居中 SVG 圆角卡（112×96），浅灰线稿（文档 + 放大镜），描边 `--border-strong`，无填充噪点 |
| 主文案 | 「词库还是空的」18/600 `--text-primary`，居中 |
| 副文案 | 「点击下方按钮添加你的第一个词条」13 `--text-secondary` |
| 主操作 | 「+ 新增词条」按钮，--accent 底白字，圆角 8，高 38（触控区 ≥44），hover 上移 1px + 阴影 |
| 无障碍 | 插画 `role="img"` + `aria-label`；按钮焦点环 2px `--accent`；空态区 `aria-live="polite"` 告知屏幕阅读器 |

### 8.2 加载态 Loading（进入 / 刷新词库）
| 元素 | 规格 |
|---|---|
| 触发 | 进入页面或点「刷新」时，表格区显示骨架屏，数据就绪后淡出 |
| 骨架行 | 高度 44，圆角 6，底色 `--bg-elevated`；上覆 shimmer 高光（`@keyframes sh` 1.2s 循环，左→右渐变扫光） |
| 工具栏 | 搜索框 / 按钮 `disabled`（opacity .6，`pointer-events:none`），防止操作未完成数据 |
| 状态栏 | 左侧旋转 spinner（16px，`@keyframes rot` .8s linear）+ 文字「加载中…」`--text-secondary` |
| 过渡 | 骨架淡出 + 内容淡入（`transition .2s ease`），尊重 `prefers-reduced-motion`（关闭 shimmer/旋转） |

### 8.3 批量选择态 Batch（勾选 ≥1 行）
| 元素 | 规格 |
|---|---|
| 触发 | 勾选任一行复选框，或点「批量管理」/ 表头「全选」 |
| 批量条 | 高 48，背景 `--accent-soft`（浅 #E6F0F9 / 深对应令牌）；左「已选 N 项」14/600 `--accent`，N 实时计数 |
| 批量操作 | 批量删除（危险红文字按钮）/ 导出 CSV（幽灵描边）/ 取消选择（文字按钮）；按钮高 ≥36，间隔 8 |
| 行复选框 | 20×20 圆角 5，点击热区 28；选中 `.on` = `--accent` 底 + 白勾；表头「全选」勾选框同步 |
| 选中行 | 背景 `--accent-row`（浅 #EFF5FB）；左侧 3px `--accent` 强调条；hover 仍保持选中底色 |
| 退出 | 「取消选择」清空所有勾并隐藏批量条；计数归零 |
| 无障碍 | 批量条 `role="status"` `aria-live="polite"` 播报「已选 N 项」；复选框原生 `QCheckBox` 保证键盘可达 |
