# RimeConfig 0.2.0 · 界面风格方案概览

> 日期：2026-07-10 ｜ 设计者：UI Designer ｜ 交付：3 张概念图（PNG）

## 已完成

根据项目当前功能（词库管理、符号表、方案配置、设置、系统托盘、热键收藏、冲突高亮、自动部署），设计了 RimeConfig 0.2.0 主界面（词库管理页）的 3 种视觉风格方案，并以 1180×760 像素 PNG 渲染交付。

## 风格方案

| 方案 | 风格名称 | 核心特征 | 适用场景 |
|---|---|---|---|
| A | Fluent Light | 类 Windows 11 原生感，浅色基底，#185FA5 品牌蓝，圆角 8px，清爽可读 | 偏爱系统原生、白天高频使用 |
| B | Dark Pro | 炭灰深色，#4A9EFF 高亮蓝，紧凑专业，左侧激活项带蓝色指示条 | 暗色环境、长时操作、与现有 dark 主题一致 |
| C | 水墨 · 文人雅致 | 暖纸色调 F5F1E8，黛青 #2F5D50 主色，朱砂 #B23A2E 印章点缀，中文宋体标题 | 追求中文输入法工具的文化辨识度、桌面个性化 |

## 方案共同点（便于对比）

- 统一布局：左侧导航栏 + 顶部工具栏 + 词条表格 + 底部状态栏，公平比较视觉风格。
- 展示功能：搜索框、+新增、分组/排序下拉、表格列（汉字 / 拼音 / 权重 / 分组 / 操作）、编码冲突高亮（#FFF7E6 或对应主题色）、状态栏统计、已连接 Rime 状态。
- 控件尺寸：按钮 36px 高（满足 WCAG 2.1 最小 44px 触控目标），正文字号 12–13px，表格行 10px 内边距，保证可读性。

## 设计决策

- 以「词库管理」作为英雄页面（主屏），因为 custom_phrase.txt 是用户最高频操作对象。
- 三个方案只在色彩、质感、字体气质上做区分，结构保持一致，避免同时比较交互差异。
- 冲突行高亮沿用了现有 `#FFF7E6`（浅主题）和暖黄底（深主题），不引入新语义色。
- 水墨方案使用「輸」字印章，提示用户这是输入法配置工具；但也会提示：若用户没有合适的宋体字体，可降级为微软雅黑。

## 后续步骤

1. 用户确认一个风格方向（A / B / C）。
2. 沿确认方向扩展完整设计系统：符号表、方案配置、设置、快速收藏弹窗、托盘菜单、深浅色双主题（如选 A 或 C）。
3. 输出 QSS 设计令牌、组件规范、间距/字号/圆角体系，便于后续开发落地。
