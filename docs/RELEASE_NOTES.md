# 本地发布记录

## v0.6.4 - 2026-07-16

### 自启动与更新

- 开机自启动改为校验快捷方式目标、`--autostart` 参数与当前程序路径；设置页和托盘勾选状态以实际检测结果为准，移动程序后可识别并修复失效自启项。
- 应用内更新按当前包类型选择 GitHub Release 资产：单文件包使用 `RimeConfig.exe`，目录式包使用 `RimeConfig-portable.zip`。
- 目录式更新改为在用户配置目录暂存、验证完整目录、退出后整体交换目录并支持回滚，避免目录包被错误替换成单个 EXE 后发生 DLL 缺失。

### 备份命名与恢复兼容

- 新建备份改为“原文件名_日期时间.bak”规则，例如 `custom_phrase.txt_20260716_153045.bak`；文件从备份目录单独取出时仍可辨认来源。
- 同一秒内重复备份自动追加序号，避免覆盖已有备份。
- 保留对历史短时间戳备份及早期平铺备份的识别、恢复与自动清理兼容性；不自动重命名旧备份。
- 恢复操作仅接受当前受管文件的已识别备份，防止误将其他文件作为恢复来源。

### 目录式发布包

- 修复目录式发布包在移动、更名或更换保存位置后，主题样式与图形界面资源无法正确加载的问题。
- 勾选图标改为 Qt 内置资源，主题样式不再拼接发布目录的绝对路径；复制到中文、空格或特殊字符路径后仍可完整解析。
- 增加 Qt 原生调色板回退，避免目录包在 Windows 外观差异下出现弹窗黑底黑字；托盘统一使用内置多尺寸图标，避免外部图标路径导致黑色图标。
- 复选框与表格勾选改为主题专用的完整“色块加白勾”图标，禁用态使用灰色图标，避免仅显示细小勾选符号。
- 新增冻结程序迁移至中文路径的主题渲染回归测试，覆盖弹窗背景、复选框与样式资源解析。

### 验证

- 自动化测试：`146 passed`；测试环境保留一条 `.pytest_cache` 无写权限警告，不影响程序功能。

## v0.6.2 - 2026-07-15

### 界面与重码维护

- 重码浏览窗口和二级集中编辑窗口提高默认高度，表格区域随窗口扩展，减少首次打开后手动调整的需要。
- 设置页 GitHub、备份与恢复区的空白状态行改为仅在出现提示时显示。

- 水墨主题移除会覆盖左上内容的朱砂印章，侧栏检索与分组整体向内留白。

### 运行优化与发布

- 候选条针对同一编码复用受管词权重映射，避免引擎状态刷新时重复扫描词库。
- 单实例唤醒轮询由 250ms 调整为 400ms，降低常驻空转。
- 更新用户说明，发布单文件 `RimeConfig.exe` 与目录式 `RimeConfig-portable.zip` 两种 Windows 包；应用内更新继续使用单文件 EXE。
- 自动化验证：`130 passed`；测试环境保留一条 `.pytest_cache` 无写权限警告，不影响功能。

## v0.6.1 - 2026-07-15

### 界面与主题

- 多编码编辑、采集、单条编辑、删除确认、重码与维护弹窗统一为分区工作台布局；多编码窗口以已确认视觉图为基准。
- 浅色、深色、水墨主题统一使用同一字体与字号体系；复选框和滚动条提高对比度与可辨识度。
- 收窄词库操作列，修复侧栏搜索框边框，并在 GitHub 区域增加用户说明与更新说明入口。

### 备份与发布

- 备份设置重排为紧凑的保留份数、周期、开关和操作区；显示下次预计备份日期并支持直接打开备份目录。
- 发布版默认备份至用户文档下的应用目录，已有自定义路径保持不变；备份改为短日期时间名称。
- 后续目录式发布包固定为 `dist/RimeConfig/`，单文件继续为 `dist/RimeConfig.exe`，均不携带版本号。
- GitHub 区的版本、仓库、用户说明、更新说明与检查更新采用同一自适应行；窗口不足时保留右侧更新操作。
- 备份区改为右侧等宽操作列：保存设置、立即备份、恢复所选备份逐行对齐；预计下次备份日期独立显示。
- 多编码编辑窗口改用固定五列候选带和专用分区样式，避免通用弹窗样式压缩候选、编码和状态行。
- 搜索框与排序框使用相同可用宽度策略，确保边框完整。
- 自动化验证：`130 passed`；测试环境保留一条 `.pytest_cache` 无写权限警告，不影响功能。

## v0.6.0 - 2026-07-14

本次已生成本地单文件 EXE，未上传 GitHub。

### 本轮性能优化

- 候选预览进程在连续编辑时保持预热，90 秒无操作后自动释放；程序明确退出时立即结束。
- 符号表删除操作改为轻量文本单元格，避免为每一行创建独立按钮控件。
- 符号表和设置页改为首次打开时再初始化，缩短主窗口首屏准备时间。
- 多编码编辑中的拼音分隔显示加入结果缓存，减少重复推断。
- 分组计数复用词库页全量缓存，避免重复复制整份词条列表。

### 运行与资源

- 系统词典索引改为流式读取并直接分批写入 SQLite，降低大词典索引时的峰值内存。
- Rime 候选预览由界面固定轮询改为服务事件驱动，减少空闲定时器与不必要的界面刷新。
- 词条备注、标签、分组规则及操作历史写入应用本地数据目录，不污染 Rime 用户目录和部署内容。

### 词库维护

- “显示重码字符/文本”改为独立全词库管理窗口，可按同文本多编码或同编码多文本查看；双击后可集中编辑文本、编码和权重，并执行一次备份、一次保存和一次部署。
- 新增词库检查、导入导出、修改历史、备注标签、自定义分组规则与快捷筛选入口。
- 批量删除、批量调整分组成功后自动退出批量选择状态，清空勾选并收起选择列。
- 分组名称后显示当前条目数。

### 本地产物

- 单文件可执行文件：dist/RimeConfig.exe。
- 目录式发布包：dist/RimeConfig-0.6.0-dir/。
- 本次仅本地打包，不上传 GitHub。

### 验证

- 自动化测试：128 passed。
- 已知测试环境提示：.pytest_cache 无写权限会产生一条警告，不影响程序功能。

## v0.4.3 - 2026-07-14

### 启动、托盘与自启

- Windows 开机自启快捷方式使用专用 `--autostart` 参数：登录后仅托盘常驻，普通双击 EXE 或开始菜单启动仍显示主界面。
- 重复启动不再静默退出；第二个进程会通知已运行实例恢复、置前并聚焦主窗口。
- 托盘图标兼容 Windows 未派发双击事件的情况：延迟处理触发事件，同时保留双击优先恢复窗口。

### 文件与更新可靠性

- 词库、设置、分组和拼音显示 sidecar 改为临时文件写完后原子替换，写入异常时保留旧文件。
- 自动更新下载时校验 GitHub 提供的 SHA-256 摘要；交接程序会等待旧进程退出、保留备份、验证新进程启动，并在失败时恢复旧版本并写入更新日志。

### 启动资源

- 系统词典索引与 Rime 候选预览引擎不再早于主窗口构建；普通启动在首帧后后台预热，开机自启则等用户首次打开界面再启动。

### 验证

## v0.4.0 - 2026-07-13

本版本为本地 Windows 发布包，不上传 GitHub。

### 词库与编辑

- 支持立即备份、定期备份、保留份数和自动清理旧备份。
- 备份和恢复覆盖受管词库、输入方案、符号表及 `pinyin_display.ini`。
- 采集和编辑窗口支持一个文本维护多个编码，含候选编码的新增、移除和权重调整。
- 英文词条提供小写、大写和原样编码建议；可选择将输出文本转为大写而保持编码小写。
- 多编码删除支持批量勾选，收藏、编辑、删除均保留窗口状态提示与右下角通知。

### 热键、通知与运行

- 修复热键未选中文本时误采集程序工作目录的问题，Win32 与 keyboard 后端均覆盖路径变体。
- 热键采集、自动部署、检查更新和下载更新均移出界面主路径。
- 新增应用内托盘通知补偿，通知展示文本和编码并可手动关闭。
- 新增 Windows 单实例锁，重复启动不会再打开第二个窗口。

### 界面与性能

- 备份设置集中在“备份与恢复”；所有下拉框和数值框均需点击后才响应滚轮。
- 完成浅色、深色和水墨主题的统一语义颜色。
- 词库搜索增加防抖，分组和筛选避免重复整库排序。
- 拼音显示推导加入内存缓存；符号表批量刷新减少布局开销。
- 修正扩展词库统计：英文输入文件 4 个，中英混合用户词库 7 个，不再把脚本或源码计入。
- 日志采用 1MB 轮转并保留 3 份，避免长期运行持续增长。

### 验证与产物

- 自动化测试：`110 passed`。
- 已知测试环境提示：`.pytest_cache` 无写权限会产生一条警告，不影响程序功能。
- 本地可执行文件：`dist/RimeConfig.exe`。
