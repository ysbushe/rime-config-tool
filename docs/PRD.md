## 项目信息
- 项目名：`rime_weasel_config_tool`（snake_case）
- 形态：Windows 桌面 GUI 绿色小程序（个人使用）
- 语言：简体中文
- 默认技术栈：Python + PySide6 + keyboard + pypinyin（PyInstaller 单文件 exe）—【推断/待确认】
- 原始需求复述：为个人用户做一款管理 RIME 小狼毫 3 个配置文件的桌面小工具，解决现有脚本"需手动部署、路径硬编码、无可视化管理"等痛点；少数多音字场景由用户手动修正编码。

## 1. 产品目标
为个人 RIME 小狼毫用户提供一个**常驻系统托盘、低占用、开机自启的绿色桌面小程序**，把 `custom_phrase.txt`、`rime_frost.schema.yaml`、`symbols_v.yaml` 三个文件的维护从"手改文本 + 手动部署"升级为可视化操作；首版重点解决写完需手动重新部署（一键/写后自动部署），并继承"全局热键选中即收藏"的便捷采集方式；多音字不做自动校正，少数情况由用户手动改编码。

## 2. 用户故事
1. As a 个人输入法用户，I want 选中任意软件里的文字按热键快速入库并默认生成全拼编码，so that 我不必手动复制，少数读音差异可自行改编码。
2. As a 词库维护者，I want 在窗口里搜索/排序/增删改词条并直接看到拼音与权重，so that 我能像用表格一样管理自定义短语。
3. As a 配置调整者，I want 在界面里切换方案开关（如简繁、模糊音）和增删符号表条目，so that 我不必打开 YAML 去冒险手改。
4. As a 怕麻烦的用户，I want 改完配置一键或自动重新部署并常驻托盘，so that 改动即时生效且不打断我的工作流。
5. As a 谨慎用户，I want 修改前自动备份并能回滚，so that 改错配置能快速恢复。

## 3. 需求池（P0 / P1 / P2）

### P0 — Must have（首版必做）
- **P0-1 默认全拼 + 手动修正**：收藏/编辑时可生成无声调全拼；不做多音字候选弹窗，少数读音差异由用户手动改编码。
- **P0-2 一键自动部署**：保存文件后调用 Weasel 部署器重新部署；提供"立即部署"入口与"写后自动部署"开关。🔴【已确认优先级】
- **P0-3 词库可视化管理（基础）**：`custom_phrase.txt` 条目 增/删/查/改 + 关键词搜索 + 按拼音/权重排序。
- **P0-4 绿色单文件 EXE**：PyInstaller `--onefile`，免安装、低内存常驻。
- **P0-5 系统托盘常驻**：启动/最小化/关闭窗口均驻留右下角托盘，含托盘菜单。
- **P0-6 Rime 目录自动探测**：默认 `%APPDATA%\Rime`，支持手动覆盖并记忆。

### P1 — Should have（应当做）
- **P1-1 全局热键收藏**：默认 `Ctrl+Alt+Q` 选中即复制入库并预填默认全拼；可开/关。
- **P1-2 词库分组**：UI 分组视图 + 数据落地方案（见第 5 节待确认）。
- **P1-3 权重编辑**：界面调整权重并说明语义（权重越大越靠前）。
- **P1-4 开机自启**：Startup 目录快捷方式，设置页开关。
- **P1-5 symbols_v.yaml 可视化管理**：分类 + 符号条目 CRUD。
- **P1-6 rime_frost.schema.yaml 受限配置**：switches / 常用 key_bindings / 是否启用 custom_phrase（字段范围见第 5 节待确认）。
- **P1-7 写前自动备份**：保存前备份被改文件，可回滚近 N 份。

### P2 — Nice to have（可选）
- **P2-1 冲突/重复检测**：同音同字高亮提示，不再用弹窗打断。
- **P2-2 深色/浅色主题**。
- **P2-3 部署失败提示与日志**。
- **P2-4 词条 CSV 导入/导出**。
- **P2-5 全局热键自定义（非默认组合）**。

## 4. UI 设计稿描述

### 主窗口（Tab 布局）
```
┌──────────────────────────────────────────────────────────────┐
│  RIME 配置小工具            [最小化] [关闭→托盘]              │
├──────────┬───────────────────────────────────────────────────┤
│ [词库管理]│  (词库管理页)                                    │
│ [符号表]  │  [🔍搜索______] [+新增] [分组:全部▾] [排序:权重▾]│
│ [方案配置]│  ┌────────────────────────────────────────────┐ │
│ [设置]    │  │ 汉字 | 拼音 | 权重 | 分组 | 操作           │ │
│          │  │ 银行 | yinhang | 100 | 工作 | ✎ 🗑       │ │
│          │  │ 你好 | nihao  | 50  | 生活 | ✎ 🗑       │ │
│          │  └────────────────────────────────────────────┘ │
└──────────┴───────────────────────────────────────────────────┘
```
- **词库管理页**：左分组树 + 右表格 + 顶部搜索/筛选；双击行或"+"开编辑弹窗。
- **符号表页**：分类列表 + 符号条目表格（键位 → 符号数组），支持增删改。
- **方案配置页**：只读元信息（方案名/版本）+ 开关区（switches 复选）+ key_bindings 列表 + "启用 custom_phrase" 开关。
- **设置页**：Rime 目录、开机自启、全局热键、自动部署、备份份数、主题。

## 5. 待确认问题（需用户答复）

**Q1. schema / symbols 具体管理字段**（❗关键，影响 P1-5 / P1-6）
- `rime_frost.schema.yaml` 建议只做**受限配置**而非全量 YAML 编辑（避免改坏）：
  - 只读：schema 名称 / 版本 / 作者
  - 开关：`switches`（如 简繁、全/半角、模糊音等，按现开关项渲染复选框）
  - 列表：`key_bindings` 常用项 CRUD（翻页 / 选重等）
  - 开关："是否启用 custom_phrase"（控制 `engine/translators` 里的 custom_phrase 引用）
- `symbols_v.yaml` 建议做：分类 + 符号条目 CRUD（键位 → 符号数组）。
- 👉 需用户确认：是否同意"受限配置"思路？要暴露哪些具体 switches / key_bindings？

**Q2. 全局热键默认值与可配置性**
- 建议默认沿用 `Ctrl+Alt+Q`（与现有脚本一致，用户已习惯）。
- 👉 是否保持默认？是否需 P2 热键自定义？首版先做"可开/关"即可。

**Q3. 词库"分组"如何落地**（custom_phrase 原生不支持分组）
- 方案 A（推荐）：UI 侧维护分组视图，分组作为元数据存于 sidecar `.json`/`.db`，`custom_phrase.txt` 仍保持 RIME 原生格式（可读性优先、部署零风险）。
- 方案 B：用 RIME 原生 `group`/括号标签写入 `custom_phrase.txt`（分组被引擎识别，但格式复杂、易错）。
- 方案 C：拆分独立分组文件（破坏单文件约定）。
- 👉 请用户选 A/B/C；我倾向 A，零风险且不影响 RIME 解析。

**Q4. 技术栈评估与确认**
- 推荐 **Python + PySide6 + keyboard + pypinyin，PyInstaller 单文件 exe**。
- 替代方案：AHK v2 单语言 / Go + Fyne / .NET。
- 👉 请确认采用 Python 方案或指定替代。

**Q5. 部署触发机制的可靠性**（技术风险，需本机验证）
- 计划调用 Weasel 部署器（`WeaselDeployer.exe /deploy`）。
- 👉 需在本机验证静默触发；若不行降级为"一键打开托盘重新部署"。

**Q6. 备份/回滚策略**（关联 P1-7）
- 建议：每次写文件前复制原文件到 `Rime\.backup\`，保留近 N 份（默认 5），提供回滚列表。

**Q7. 冲突/重复行为**（关联 P2-1）
- 建议首版改为列表内高亮 + 静默 +1 权重（保持现有"拼音冲突 +1 权重"语义）。
