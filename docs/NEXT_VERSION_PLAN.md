## 已完成：系统词典候选提示与权重建议

### 目标

在采集窗口中检测系统词典是否已存在同文本、同编码的候选；若命中，展示词典来源、参考原始权重与本工具参考优先级，并允许用户采用高于参考值的自定义权重。

### 实现

- 不在采集时扫描 Rime 编译词库或大 YAML 词库。
- 程序启动和采集窗口打开时，在后台解析当前 Frost 静态词典源文件，建立工具自身的 SQLite 索引。
- 索引键为文本和规范化编码，记录来源、原始权重、本工具参考优先级及词典文件签名。
- 检测到词典文件签名变化后重建索引；查询阶段只做精确索引查询。
- 索引未完成时，采集功能照常可用，并显示“系统词典索引准备中”。

### 当前配置依据

- 主系统词典：`rime_frost`，本工具参考优先级 `1.2`。
- 英文词典：`melt_eng`，本工具参考优先级 `1.1`。
- 中英混合用户词典：`cn_en`，本工具参考优先级 `0.5`。
- 本工具管理的自定义短语：`custom_phrase`，本工具参考优先级 `99`，通常已具有较高候选优先级。

### 边界

- 静态索引可准确提示“系统词典已有同词同码”及其原始权重。
- 不承诺完全复现输入法实时第一候选，因为最终排序还会受过滤器、学习记录和翻译器质量影响。

## 已完成：v0.4.3 稳定性修复

### 自动更新交接可靠性

- 现象：本地 `v0.4.1` 从设置页升级至 `v0.4.2` 后，新 EXE 在 PyInstaller 解包阶段提示无法加载 `%TEMP%\_MEI...\python313.dll`。
- 已确认：GitHub 发布资产 SHA-256 与本地验收包一致，问题发生在下载完成后的替换与重启交接，而非发布文件损坏。
- 修复方向：更新脚本按旧进程 PID 等待完全退出；校验下载文件 SHA-256；先保留 `.bak` 再原子替换；只在替换成功后启动新程序；失败与启动异常均写入日志并允许回滚。
- 兼容说明：旧 `v0.4.1` 更新器不能获得此修复，需手动安装一次后续版本。

### 托盘图标双击恢复窗口

- 现象：Windows 上双击托盘图标无法恢复主窗口。
- 当前实现已监听 `QSystemTrayIcon.DoubleClick`，但需实机核验激活原因；下个版本应补充 `Trigger` 回退处理与窗口 `showNormal / raise / activate` 的可观察验收。
- 验收：主窗口隐藏或最小化至托盘后，双击托盘图标能稳定恢复并置前；右键菜单行为不受影响。

### 开机自启的窗口行为

- 现象：用户希望 Windows 登录后程序仅在托盘常驻，不抢占桌面；之后从托盘菜单或双击图标打开时，必须正常显示主界面。
- 修复方向：自启快捷方式带专用启动参数，例如 `--autostart`；程序仅在收到该参数时于初始化完成后隐藏至托盘，普通双击 EXE 与开始菜单启动始终显示主界面。
- 验收：开机启动不弹出主窗口；托盘“打开”、双击图标和已有实例后的再次启动请求均能恢复、置前并聚焦主窗口。

## 后续功能升级候选

### 词库健康检查

- 一键检查空文本、空编码、重复项、非法权重、重码、无效分组及可能不生效的 Rime 配置。
- 检查结果可定位到条目，并按用户确认逐项或批量修复。

### 恢复前差异预览

- 恢复备份前，分别展示词库、符号表、方案配置的新增、修改与删除项。
- 支持只恢复一项受管文件，避免整份备份覆盖无关的近期修改。

### 持久化修改历史

- 记录新增、编辑、删除、批量分组和恢复操作；关闭程序后仍可查看近期记录。
- 支持按记录恢复，降低误删和错误批量操作的风险。

### 部署状态中心

- 显示最近部署时间、成功或失败状态、Rime 返回信息和受影响文件。
- 提供重新部署入口，并在失败时给出可操作的排查提示。

### 词条冲突决策

- 采集或编辑时汇总当前文本是否已有自定义词、系统词典来源、静态权重、建议自定义权重及预计候选位置。
- 用户可据此明确选择新增、覆盖或调整权重。

### 组合查询与筛选

- 支持按编码查候选、未分组、英文、低权重、最近修改等条件组合筛选。
- 词库规模增长后仍能快速定位需要处理的条目。

### 自定义分组规则

- 在现有自动分组基础上，允许用户定义文本、编码前缀或类型的归类规则。
- 应用前提供预览，确认后一次性批量执行。

### 导入导出与迁移

- 支持导入常见 TSV 或文本词库；写入前预览冲突、编码格式和将产生的修改。
- 支持导出受管词库与分组信息，便于迁移和分享。

### 词条备注与标签

- 为词条保存不写入 Rime 的本地备注、来源、用途和标签。
- 支持按标签查询与批量管理，不污染输入法词库内容。

- 自 `v0.5.0` 起，每次新增、调整或合并后续功能后，先重新审核数据规模、调用频率、界面渲染、磁盘写入、后台线程和常驻进程，再确定是否优化及应在哪一版本实施。
- `v0.5.0` 数据安全功能：修改历史使用追加式轻量记录与定期压缩；恢复差异和健康检查采用按需、后台、可取消扫描，避免启动时遍历全部备份。
- `v0.5.1` 管理效率功能：重码使用可失效的全库索引；分组计数与组合筛选复用预聚合统计；重码和批量编辑弹窗使用模型视图或分页，避免为全部条目创建控件。
- `v0.6.0` 导入导出功能：采用流式读取、分批校验和单次事务式写入，避免大型词库导入占用过多内存或触发多次部署。
- 每个阶段实现前先记录基线，完成后用同一指标复测；未出现可感知瓶颈时优先保持实现简单，不引入无收益的常驻缓存或后台服务。

### 阶段交付门槛

- 每阶段均须完成聚焦单测和完整 pytest、拉起 GUI 逐项验收，并更新版本说明。
- 后续默认不打包 EXE；只有用户明确要求时才生成单文件或目录式发布包。
- GitHub 发布须在用户明确确认后执行。
